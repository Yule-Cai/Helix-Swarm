AUTONOMOUS MULTI-AGENT SWARM
A swarm that acts. A helix that keeps it under control.
Swarm 负责行动，Helix 负责控制。

Helix-Swarm is a permission-first CLI agent framework. One strand pushes agent work forward; the other checks permission, evidence, and traceability before risky actions happen.

Helix-Swarm 是一个权限优先的 CLI Agent 框架。一条链推动 Agent 工作； 另一条链在高风险动作发生前检查权限、证据和可追踪性。

Meaning
命名含义

Helix controls. Swarm collaborates.

Helix 负责控制，Swarm 负责协作。

The name comes from the original vision: one agent plans the whole task, multiple agents work together, and a second strand keeps permissions and evidence attached to every action.

这个名字来自最初的设想：一个 Agent 进行整体规划，多个 Agent 协同执行， 同时另一条链把权限和证据绑定到每个动作上。

01

Double-helix architecture

双螺旋架构

One strand is agency: routing, agents, tools, and execution. The other is control: permission, evidence, audit, and approval.

一条链是能力：路由、Agent、工具和执行。另一条链是控制：权限、证据、审计和确认。

02

Swarm-style delegation

Swarm 式委派

Leo supervises specialist agents such as File Agent, Search Agent, Reviewer, and Computer Agent.

Leo 监督 File Agent、Search Agent、Reviewer 和 Computer Agent 等专家 Agent。

03

Control at every crossing

每个交叉点都有控制

When an action becomes risky, the control strand crosses the agency strand and asks for approval.

当动作变得高风险时，控制链会穿过能力链并请求用户确认。

Features
核心能力

Controlled by design, not blind autonomy.

以可控性为核心，而不是盲目自动化。

Helix-Swarm focuses on the agent behaviors that matter most: explain actions, ask before risk, show evidence, and keep work traceable.

Helix-Swarm 聚焦 Agent 最关键的行为：解释动作、高风险前确认、展示证据、保持可追踪。

A1

Permission-first execution

权限优先执行

Risky operations are reviewed before they run. The agent does not silently execute high-impact actions.

高风险操作执行前先审查，Agent 不会静默执行高影响动作。

A2

Human-in-the-loop approval

人在回路确认

Approve, deny, or block tool calls. The user keeps final authority over sensitive steps.

用户可以确认、拒绝或禁止工具调用，保留对敏感步骤的最终决定权。

A3

Evidence Card review

Evidence Card 审查

Review findings must include file, symbol, evidence, risk, consequence, and suggested fix.

审查发现必须包含文件、函数、证据、风险、后果和修复建议。

B1

Skill-aware workflows

技能感知工作流

Search, compare, inspect, and then decide. Skills are not blindly installed or invoked.

先搜索、比较、检查，再决定是否安装或调用 Skill。

B2

Terminal control layer

终端控制层

A CLI-first interface for chatting, reviewing tools, changing settings, and supervising actions.

CLI 优先，用于聊天、审查工具、修改配置和监督动作。

B3

Traceable agent actions

可追踪 Agent 行为

Tool calls, permission checks, and review outputs are visible instead of hidden inside the model.

工具调用、权限检查和审查输出都保持可见，而不是隐藏在模型内部。

Agent Swarm

Built-in agents with clear boundaries.

内置 Agent，边界清晰。

Helix-Swarm is not one giant agent. It is a small swarm coordinated by Leo, where every specialist has a clear role, input, output, and safety boundary.

Helix-Swarm 不是一个巨大的单体 Agent。它是由 Leo 协调的小型 Swarm， 每个专家 Agent 都有明确职责、输入、输出和安全边界。

Architecture
架构

Agency and control in one traceable system.

能力链与控制链，组成一个可追踪系统。

Leo supervises specialist agents. The registry discovers tools. The permission manager decides what needs approval.

Leo 监督专家 Agent，注册表发现工具，权限管理器判断哪些动作需要确认。

Agency Strand
User intent, task routing, specialist agents, tool proposals, and execution.
用户意图、任务路由、专家 Agent、工具提议和执行。
Control Strand
Permission review, risk estimation, evidence checks, user approval, and audit logs.
权限审查、风险判断、证据检查、用户确认和审计日志。
Swarm Layer
File Agent, Search Agent, Reviewer, Computer Agent, Tool Registry, and SOP-style skills.
File Agent、Search Agent、Reviewer、Computer Agent、工具注册表和 SOP 风格技能。
Trace Layer
Visible tool calls, approval state, execution output, and evidence-based review cards.
可见工具调用、确认状态、执行输出和基于证据的审查卡片。

Roadmap
路线图

Toward the double-helix swarm.

走向双螺旋 Swarm。

The current CLI is the control layer. The next step is stronger planning, better file pipelines, and a closer implementation of the original planner-plus-parallel-swarm vision.

当前 CLI 是控制层。下一步是更强规划、更好的文件管线，以及更接近最初“规划器 + 并行 Swarm”的设想。

01Permission-first execution权限优先执行
02Evidence Card review modeEvidence Card 审查模式
03Skill-aware workflows技能感知工作流
04Terminal control layer终端控制层
05Stronger reviewer verifier更强的审查验证器
06Double-helix planner and executor双螺旋规划与执行器

Make agent actions visible.

让 Agent 的动作保持可见。

Helix-Swarm is experimental, but its direction is clear: agent swarms should be powerful, while their actions remain reviewable.

Helix-Swarm 仍处于实验阶段，但方向很清楚：Agent Swarm 可以强大，但它的动作必须可审查。
